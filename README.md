## Technical Constraints & Context
As part of the 42 Network pedagogy, these projects adhere to strict rules:
- **C++17 Standard:** The modules are implemented using modern C++17 to align with current industry practices.
- **No standard containers** (like `<vector>` or `<list>`) are allowed until the specific STL modules.
- **No memory leaks.** Every allocation is strictly tracked and freed.
- Compilation is done using `Makefile` with `-Wall -Wextra -Werror` flags.
- Code is written following the Orthodox Canonical Form for classes.
